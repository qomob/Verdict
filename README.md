# Verdict

> AI 诉讼压力测试系统 — 通过红蓝对抗 + 法官居中裁判六阶段架构，提前发现案件漏洞并提升胜诉概率。

## 这是什么

Verdict 是一套基于**红蓝对抗 + 法官居中裁判**六阶段架构的诉讼风险分析系统。不是寻找支持结论的理由，而是尽可能发现案件中的漏洞、矛盾、风险和败诉点。

核心理念：**很多案件不是输在法律条文，而是输在证据链断裂、时间线矛盾、证明力不足、举证责任分配错误。** 本系统通过红队模式优先挖掘这些致命问题，再反向生成补强策略。

v3.0 新增三项内嵌能力：**合同条款提取、法律检索、法律文书生成**，无需独立模块，作为诉讼推演管道的自然环节运行。

## 架构

```
阶段1 → 阶段2 → 阶段3 → 阶段4 → 阶段5 → 阶段6
证据地图   红队攻击   蓝队反击   法官裁判   补强方案   终局评估
+合同提取  +法律检索                      +文书生成
```

| 阶段 | Agent | 职责 | 内嵌能力 |
|------|-------|------|---------|
| 1. 证据地图 | evidence-mapper | 建立证据全景图，标记断链/孤证/循环论证 | 合同条款提取 |
| 2. 红队攻击 | red-team-lawyer | 站在被申请人立场，穷尽手段推翻申请人主张 | 法律检索 |
| 3. 蓝队反击 | blue-team-lawyer | 逐条反驳红队全部攻击，给出法律依据和成功概率 | — |
| 4. 法官裁判 | judge | 居中评价双方表现，给出证据采信度和胜诉概率 | — |
| 5. 补强方案 | reinforcement-advisor | 制定 P0/P1/P2 三级证据补强方案 | 法律文书生成 |
| 6. 终局评估 | risk-assessor | 输出危险漏洞TOP10、败诉论点TOP10、风险评级 | — |

### 内嵌能力说明

| 能力 | 所在阶段 | 作用 |
|------|---------|------|
| 合同条款提取 | 阶段1 | 自动识别合同/协议文本，逐条提取分类（标的/价款/期限/违约/管辖/解除/附随），作为独立证据项纳入证据地图 |
| 法律检索 | 阶段2 | 攻击前按法律→司法解释→指导案例→典型判例→裁判规则五层检索，为每个攻击点提供法条支撑 |
| 法律文书生成 | 阶段5 | 根据审理阶段自动生成起诉状/答辩状/质证意见/代理词草稿，引用攻防分析中的证据编号和法条 |

这三项能力不是独立模块，而是嵌入在诉讼推演管道的对应阶段中，与对抗博弈流程共享数据。

## 适用场景

- 民事诉讼庭前准备
- 劳动仲裁案件分析
- 合同纠纷风险审查
- 证据整理与质证准备
- 案件风险评估与决策
- 模拟庭审训练
- 法律文书起草
