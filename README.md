# Verdict | 裁决

> AI 诉讼操作系统 — 在起诉、应诉、庭审和和解之前，先让 AI 扮演对方律师、法官和上诉法官，把案件打一遍。

## 这是什么

Verdict 是一套基于**中立检索→红蓝对抗→法官推演→和解策略**十阶段架构（阶段-1 接案门控 + 阶段0-8）的诉讼风险分析系统。不是寻找支持结论的理由，而是尽可能发现案件中的漏洞、矛盾、风险和败诉点。

核心理念：**很多案件不是输在法律条文，而是输在证据链断裂、时间线矛盾、证明力不足、举证责任分配错误。** 本系统通过红队模式优先挖掘这些致命问题，再反向生成补强策略。

v4.3 架构升级：
- **七法域支持** — 中国大陆(cn) / 美国(us) / 日本(jp) / 韩国(kr) / 欧洲(eu) / 东南亚(sea) / 中东(me)，法域路由自动切换法律框架
- **接案评估门控（阶段-1）** — 时效/管辖/案由/起诉条件/案件强度 5 项检查，在推演前拦截硬伤案件
- **四入口路由** — 冷启动配置 / 完整推演 / 快速文书 / 合同审查，按需执行不浪费
- **事实中性化阀门** — 阶段-1 剥离用户输入中的法律定性词，输出 neutralized_facts，防止预设结论污染下游中立检索
- **中立法律检索（阶段0）** — 消除红队选择性偏差，为双方提供共享弹药库
- **MCP 法律检索增强** — 阶段-1/阶段0 接入 mcp_Jina_AI，search_web 检索权威来源 + read_webpage 读取原文 + fact_check 验证每条引用（去重校验 + verification_log），杜绝法条幻觉和引用已废止法律
- **请求权基础预选** — 大陆法系法域（cn/jp/kr/eu）在检索前穷尽候选请求权基础（主要规范/辅助规范/防御规范），形成攻防网络
- **比例原则（冗余即错误）** — 红队蓝队按争议程度分级展开（无争议/轻度争议/核心争议），不为凑数制造噪声攻击点
- **合同审查** — 从"条款提取"升级为 8 维风险审查（无效/歧义/举证/管辖/付款/违约/履约/撤销）
- **多维证据评分** — 真实性/合法性/关联性/证明力/可采性五维打分 + Evidence Heatmap
- **事实认定推理链** — 法官心证形成过程可视化
- **和解策略分析（阶段6）** — 95%+ 案件不会走到判决，和解区间+底牌预测+谈判策略
- **法条引用审计（阶段7）** — 回溯全流程法条引用 verification 状态，标记支撑核心论点的未验证引用
- **上诉分析（阶段8）** — 败诉后怎么办：可改判错误识别 + 二审翻盘概率 + 新证据价值 + 再审可行性
- **案件类型路由** — 劳动/合同/侵权/股权/知识产权，按需加载缩小上下文
- **阶段聚焦表** — 各阶段明确"关注什么/不关注什么"，避免上下文污染

## 四入口路由

| 路径 | 触发场景 | 执行内容 |
|------|---------|---------|
| **S: 冷启动配置** | 首次使用、配置偏好 | 法域/MCP检测/输出偏好设置 |
| **A: 完整推演** | 诉讼推演、案件分析、败诉风险 | 九阶段全流程（阶段-1→8） |
| **B: 快速文书** | 写起诉状、答辩状、代理词 | 直接生成文书草稿，跳过推演 |
| **C: 合同审查** | 合同审查、合同风险 | 直接执行合同 8 维风险审查 |

## 架构（路径A）

```
阶段-1 → 阶段0 → 阶段1 → 阶段2 → 阶段3 → 阶段4 → 阶段5 → 阶段6 → 阶段7 → 阶段8
接案评估  法律研究  证据地图  红队攻击  蓝队反击  法官推演  补强方案  和解策略  终局评估  上诉分析
门控      +合同审查                               +文书生成          可选
```

| 阶段 | Agent | 职责 | 内嵌能力 |
|------|-------|------|---------|
| -1. 接案评估 | case-intake | 时效/管辖/案由/起诉条件检查，案件强度初评 | 门控 |
| 0. 法律研究 | legal-researcher | 中立五层检索，双向标注 | — |
| 1. 证据地图 | evidence-mapper | 建立证据全景图，标记断链/孤证/循环论证 | 合同审查（8维风险） |
| 2. 红队攻击 | red-team-lawyer | 引用阶段0检索，穷尽手段推翻申请人主张 | — |
| 3. 蓝队反击 | blue-team-lawyer | 逐条反驳红队全部攻击，给出法律依据和成功概率；阶段0未覆盖的法条缺口输出 missing_legal_basis | — |
| 4. 法官推演 | judge | 多维证据评分 + 事实认定推理链 + 胜诉概率 | Evidence Heatmap |
| 5. 补强方案 | reinforcement-advisor | 制定 P0/P1/P2 三级证据补强方案 | 法律文书生成 |
| 6. 和解策略 | settlement-analyzer | 最佳和解区间 + 双方底牌 + 谈判策略 | — |
| 7. 终局评估 | risk-assessor | 输出危险漏洞TOP10、败诉论点TOP10、风险评级 | — |
| 8. 上诉分析 | appeal-analyzer | 可改判错误识别 + 二审翻盘概率 + 新证据价值 + 再审可行性 | — |

### 内嵌能力说明

| 能力 | 所在阶段 | 作用 |
|------|---------|------|
| 接案评估 | 阶段-1 | 门控：时效/管辖/案由/起诉条件检查，拦截硬伤案件 |
| 中立法律检索 | 阶段0 | 攻防开始前全面检索，双向标注（倾向申请人/被申请人/中立），消除选择性偏差 |
| 合同审查 | 阶段1 | 逐条 8 维风险审查（无效/歧义/举证/管辖/付款/违约/履约/撤销） |
| 法律文书生成 | 阶段5 | 根据审理阶段自动生成起诉状/答辩状/质证意见/代理词草稿 |
| 和解分析 | 阶段6 | 最佳和解区间计算 + 双方底牌预测 + 谈判路径设计 |
| 上诉分析 | 阶段8 | 假设败诉场景，识别可改判错误，评估翻盘概率和上诉策略 |

## 法域路由

根据用户指定的国家/地区，按需加载对应法域的 reference 文件。默认法域为中国大陆（cn）。

| 法域代码 | 法域 | 触发示例 |
|---------|------|---------|
| `cn` | 中国大陆 | 中国、大陆、境内（默认） |
| `us` | 美国 | 美国、联邦、US |
| `jp` | 日本 | 日本、Japan |
| `kr` | 韩国 | 韩国、Korea |
| `eu` | 欧洲 | 欧洲、欧盟、EU |
| `sea` | 东南亚 | 东南亚、东盟、SEA |
| `me` | 中东 | 中东、海湾、Middle East |

## 适用场景

- 多法域民事诉讼庭前准备（中国大陆/美国/日本/韩国/欧洲/东南亚/中东）
- 劳动仲裁/雇佣纠纷案件分析
- 合同纠纷风险审查
- 合同条款风险审查（独立路径C）
- 证据整理与质证准备
- 案件风险评估与决策
- 和解谈判策略制定
- 模拟庭审训练
- 法律文书起草（独立路径B）
