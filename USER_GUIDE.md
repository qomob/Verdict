## 1. 快速开始

### 步骤一：选择路径

根据你的需求选择路径：

| 你想做什么 | 选择路径 | 怎么触发 |
|-----------|---------|---------|
| 全面分析案件风险 | **路径A：完整推演** | 说"进行诉讼推演" |
| 只需要写法律文书 | **路径B：快速文书** | 说"帮我写起诉状" |
| 只需要审查合同风险 | **路径C：合同审查** | 说"帮我审查这份合同" |

### 步骤二：准备案件信息

按以下模板整理你的案件信息：

```
【案件背景】
- 当事人信息及关系
- 事件发生时间线
- 争议产生过程
- 已发生的法律程序（如仲裁、一审等）

【争议焦点】
1. ...
2. ...

【申请人证据】
- 证据1：名称 | 证明目的 | 证据形式（书证/电子数据/录音录像/证人证言/鉴定意见/其他）
- 证据2：...

【被申请人证据】（如有）
- 证据1：...

【适用法律领域】
劳动争议 / 合同纠纷 / 侵权责任 / 股权纠纷 / 知识产权 / 其他

【管辖地】（可选）
省/市

【审理阶段】（可选）
仲裁 / 一审 / 二审 / 再审
```

### 步骤三：发起请求

直接将案件信息发送给 AI，并说明你要做什么。系统会自动路由到对应路径。

## 2. 三条路径选择

### 路径A：完整诉讼推演

**适合场景：** 案件准备阶段，需要全面了解风险和胜诉概率。

**执行流程：** 阶段-1（接案评估）→ 阶段0（法律研究）→ 阶段1（证据地图）→ 阶段2（红队攻击）→ 阶段3（蓝队反击）→ 阶段4（法官推演）→ 阶段5（补强方案）→ 阶段6（和解策略）→ 阶段7（终局评估）→ 阶段8（上诉分析，可选）

**预计交互：** 多轮对话，每个阶段输出后你可以确认或追问。

### 路径B：快速文书生成

**适合场景：** 你已经清楚案件情况，只需要快速生成法律文书草稿。

**怎么用：** 提供案件信息和证据，直接说"帮我写起诉状/答辩状/代理词/质证意见"。

**输出：** 符合法院提交规范的文书草稿，包含诉讼请求、事实理由、法律依据、证据清单。

### 路径C：合同审查

**适合场景：** 签约前审查合同风险，或合同纠纷中分析条款效力。

**怎么用：** 提供合同/协议原文，说"帮我审查这份合同"。

**输出：** 逐条 8 维风险审查（无效/歧义/举证/管辖/付款/违约/履约/撤销）+ 核心风险 TOP5。

## 3. 准备案件信息

### 必填项

| 字段 | 说明 | 示例 |
|------|------|------|
| **案件背景** | 当事人信息、事件时间线、争议产生过程、已发生的法律程序 | "申请人张三2024年3月入职被申请人A公司，2024年9月被单方解除劳动合同..." |
| **争议焦点** | 核心争议，逐条列明 | "1. 解除劳动合同是否合法 2. 是否应支付经济赔偿金" |
| **申请人证据** | 编号、名称、证明目的、证据形式 | "证据1：劳动合同（书证），证明双方存在劳动关系及工资约定" |
| **法律领域** | 匹配对应专家分析 | "劳动争议" |

### 选填项

| 字段 | 作用 |
|------|------|
| 被申请人证据 | 如有，让分析更全面 |
| 管辖地 | 匹配地方性法规和裁判口径（如北京 vs 深圳的劳动仲裁口径差异） |
| 审理阶段 | 匹配对应程序规则（仲裁/一审/二审/再审的举证规则不同） |
| 合同文本 | 如涉及合同纠纷，提供合同/协议原文，系统将进行 8 维合同审查 |

### 证据信息越详细，分析越精准

好的证据描述：
> 证据3：微信聊天记录截图（电子数据），2024年8月15日至9月20日，申请人与HR李四的对话，证明公司主动提出解除，证明目的——公司系单方解除而非协商解除。原件保存于申请人手机，已截图打印。

差的证据描述：
> 证据3：聊天记录

## 4. 九阶段详解

### 阶段-1：接案可行性评估

**这一阶段会发生什么：** 系统先做快速门控检查，判断案件是否值得继续投入分析。

**系统会检查 5 项：**

| 检查项 | 拦截什么 |
|--------|---------|
| 诉讼时效 | 是否已过时效（如劳动争议仲裁时效1年） |
| 管辖 | 管辖权是否有问题（如约定管辖无效、未走仲裁前置） |
| 案由 | 主张的法律关系是否匹配事实（如把劳动关系误认为劳务关系） |
| 起诉条件 | 原告资格、被告是否明确、诉请是否具体 |
| 案件强度 | 基于已有证据的粗略初评（A-E级） |

**门控逻辑：** 如果案件有硬伤（如时效已过），系统会建议终止并说明原因。你可以选择接受建议，或坚持继续（系统会在后续分析中标注"接案评估未通过"）。

**案件强度评级：**

| 等级 | 含义 |
|------|------|
| A | 证据充分，法律依据明确 |
| B | 证据较充分，有一定风险 |
| C | 胜败参半，关键证据待补 |
| D | 证据不足，风险较高 |
| E | 严重不足，不建议诉讼 |

---

### 阶段0：法律研究

**这一阶段会发生什么：** 中立法律研究员全面检索本案适用的法律法规，为后续红蓝双方攻防提供共享的法律基础。

**系统会输出：**

- **争议焦点拆解** — 将你的争议焦点拆解为具体法律问题
- **五层检索结果：**
  1. 法律条文（如《劳动合同法》《民法典》）
  2. 司法解释（如最高院劳动争议司法解释）
  3. 指导案例（最高院发布的参照案例）
  4. 典型判例（同类案件裁判倾向）
  5. 裁判规则（地方高院指引，如提供了管辖地）

- **双向标注** — 每条法条标注倾向：
  - 🟦 倾向申请人（对你有利的法条）
  - 🟥 倾向被申请人（对对方有利的法条）
  - ⬜ 中立（需要结合事实判断）

**为什么重要：** 法律检索在中立阶段完成，避免红队"选择性检索"只找对己方有利的法条。

---

### 阶段1：证据地图构建

**你要做什么：** 提交案件信息后，检查证据地图是否准确。

**系统会输出：**
- 结构化证据清单（双方全部证据）
- 证据关系分析（哪些证据形成闭环、哪些是孤证、哪些存在断链）
- 风险标记（⚠️ 证据断链 / ⚠️ 孤证 / ⚠️ 循环论证）
- **合同审查报告（如输入包含合同）** — 逐条 8 维风险审查

**合同审查的 8 个维度：**

| 风险维度 | 审查内容 |
|---------|---------|
| 无效风险 | 是否违反强制性规定 |
| 歧义风险 | 是否存在多种解释 |
| 举证风险 | 约定是否可举证 |
| 管辖风险 | 管辖约定是否有效 |
| 付款风险 | 付款条件是否明确 |
| 违约风险 | 违约金是否合理 |
| 履约风险 | 履行标准是否可量化 |
| 撤销风险 | 是否存在可撤销事由 |

**关键确认点：**
- 证据数量和内容是否正确
- 证据关系是否遗漏
- 风险标记是否准确
- 合同审查的风险点是否遗漏

> **确认无误后，告诉 AI 继续进入阶段2。**

---

### 阶段2：红队律师攻击

**这一阶段会发生什么：** AI 切换为被申请人律师角色，穷尽一切手段攻击你的案件。

**系统会输出：**

**（1）证据三性质质证** — 对你每一份证据逐一攻击：

| 维度 | 攻击方向 |
|------|---------|
| 真实性 | 原件缺失？来源不明？时间矛盾？可篡改？单方形成？ |
| 合法性 | 非法取得？程序违法？隐私侵权？未经授权？ |
| 关联性 | 证明不了争议焦点？推理跨度过大？有替代解释？ |

每个攻击点标注严重程度：🔴高 / 🟡中 / 🟢低

**（2）事实链攻击** — 时间线漏洞、逻辑断裂、动机漏洞

**（3）最强败诉路径** — 红队构建的让你败诉的最优 5 步路径

**（4）交叉询问设计** — 30+ 个按杀伤力排序的问题（⭐1-⭐5）

**注意：** 红队直接引用阶段0的中立检索结果，不再自行检索，避免选择性偏差。

**心态提醒：** 这一阶段会让人不舒服——因为 AI 在尽全力证明你会输。这正是它的价值。忍住反驳冲动，先看清所有攻击点。

---

### 阶段3：蓝队律师反击

**这一阶段会发生什么：** AI 切换为你的律师角色，逐条回应红队的全部攻击。

**系统会输出：**

对红队每一个攻击点：
- **回应策略** — 正面回应，不回避
- **法律依据** — 具体法条编号（引用阶段0检索结果）
- **补强证据建议** — 还能补充什么证据来应对
- **成功概率** — 0-100% 的诚实评估

**重点看：**
- `success_probability` 低于 50% 的回应 — 这些是你案件的薄弱环节
- `weak_spots` — 蓝队自己也承认难以反驳的问题
- `supplementary_evidence` — 建议补充的证据，记下来后面用
- `missing_legal_basis` — 蓝队反击中发现的阶段0未覆盖的法律问题。如果非空，系统会建议你回到阶段0补充检索后重跑蓝队

---

### 阶段4：法官居中推演

**这一阶段会发生什么：** AI 切换为法官角色，不偏不倚地评价双方。

**系统会输出：**

**（1）双方律师评分**（0-100 分）

**（2）多维证据评分** — 每份证据从 5 个维度打分：

| 维度 | 含义 |
|------|------|
| authenticity（真实性） | 原件？来源可查？时间明确？ |
| legality（合法性） | 取证合法？无侵权？ |
| relevance（关联性） | 与争议焦点关联度？推理跨度？ |
| probative_value（证明力） | 对待证事实的证明程度 |
| admissibility（可采性） | 综合判断能否被采纳 |

综合分权重：真实性 25% + 合法性 15% + 关联性 25% + 证明力 25% + 可采性 10%

**（3）Evidence Heatmap** — 证据热力图，一目了然：

```
证据1 ██████████ 90
证据2 ████░░░░░░ 40
证据3 ██████░░░░ 60
```

**（4）事实认定推理链** — 法官的心证形成过程：

```
证据A（采信度85）→ 证明事实1
证据B（采信度70）→ 佐证事实1
事实1成立（认定概率80%）
  ↓
事实1 + 法律规定 → 事实2成立（认定概率65%）
  ↓
支持/不支持原告诉请
```

**（5）胜诉概率**（申请人 vs 被申请人，合计 100%）

**关键数据：**
- `win_probability.applicant` — 这是你当前的胜诉概率基准值
- 综合分低于 50 的证据 — 这些证据法官可能不认
- `reasoning_chain` — 法官如何从证据推导到结论，比一个百分比更有价值
- `judge_notes` — 法官建议进一步查明的事项

---

### 阶段5：补强方案与文书生成

**这一阶段会发生什么：** AI 回到你的律师角色，基于法官推演暴露的弱点，制定分级补强方案，并生成法律文书草稿。

**系统会输出：**

**补强方案（三级）：**

| 级别 | 含义 | 行动建议 |
|------|------|---------|
| **P0** | 决定胜负，必须补充 | 不补充大概率败诉，立即行动 |
| **P1** | 重要，强烈建议补充 | 不补充风险显著增加，尽快补充 |
| **P2** | 辅助，有则更好 | 补充后增强说服力，条件允许时补充 |

每项都会告诉你：需要什么证据、不补的后果、怎么获取。

**法律文书草稿：** 根据审理阶段自动生成最关键的文书（起诉状/答辩状/质证意见/代理词），引用攻防分析中的证据编号和法条。

**重点关注：** `projected_win_probability` — 如果补强方案全部落实，胜诉率能提升到多少。

---

### 阶段6：和解策略分析

**这一阶段会发生什么：** AI 切换为诉讼策略顾问，基于案件胜诉概率和双方弱点，分析和解的最优策略。

**现实中 95% 以上的案件不会走到判决。** 这一阶段帮助用户判断是否应该和解，以及如何谈判。

**系统会输出：**

- **和解可行性评估** — 和解 vs 继续诉讼，哪个更优
- **最佳和解区间** — 计算下限和上限，给出中间值
- **双方底牌预测：**
  - 申请人最强筹码 + 最弱环节
  - 被申请人最强筹码 + 最弱环节
- **谈判策略：**
  - 建议开价/还价位置
  - 让步路径（先让什么、后让什么）
  - 谈判破裂点（低于/高于此值不应接受）
  - 谈判时机建议
- **双方接受概率** — 申请人接受概率、被申请人接受概率

**重点看：**
- `settlement_zone` — 如果对方的预期落在这个区间内，和解就有可能
- `walk_away_point` — 你的底线，低于/高于此值不要接受
- `party_leverage` — 你的底牌和对方的底牌，知己知彼

---

### 阶段7：终局风险评估

**这一阶段会发生什么：** AI 汇总全部分析，输出最终风险报告。

**系统会输出：**

- **最危险漏洞 TOP 10** — 按危险等级排序
- **最容易被否定的论点 TOP 10** — 法官可能不买账的主张
- **最可能导致败诉的证据问题 TOP 10** — 证据层面的致命问题
- **最应优先补强的证据 TOP 10** — 补强的优先级清单
- **整体风险评级** — A/B/C/D/E 五级

**风险评级对照：**

| 等级 | 含义 | 胜诉概率 |
|------|------|---------|
| A | 胜诉概率极高 | 80% 以上 |
| B | 胜诉概率较高 | 60%-80% |
| C | 胜败参半 | 40%-60% |
| D | 胜诉概率较低 | 20%-40% |
| E | 高概率败诉 | 20% 以下 |

**最重要的一行：** `core_issue_summary` — 本案真正决定胜负的核心问题。所有分析最终收敛到这一句话。

**分析质量审查：** `analysis_quality_review` — 系统会自我审查本次推演的质量：

- **跨阶段矛盾检测** — 检查各阶段输出之间是否有矛盾（包括和解区间 vs 胜诉概率是否一致）
- **遗漏检测** — 检查是否有关键证据在某个阶段被遗漏
- **置信度趋势** — 汇总各阶段的置信度，如果后半程下降，说明分析可靠性降低
- **整体可靠性** — 为 low 时，提示本次推演结果仅供参考

---

### 阶段8：上诉分析（可选）

**这一阶段会发生什么：** 假设你败诉了，分析"如果输了怎么办"。

**触发条件：** 用户要求"上诉分析""如果输了怎么办""二审翻盘"时加载；或阶段7风险评级为 D/E 时系统会主动建议。

**系统会输出：**

**（1）可改判错误识别** — 从法官推演中找出可能被二审推翻的错误：

| 错误类型 | 说明 |
|---------|------|
| 程序违法 | 管辖错误、送达违规、质证程序缺失 |
| 法律适用错误 | 法条引用错误、解释不当、新旧法适用 |
| 事实认定错误 | 证据采信不当、关键事实遗漏、推理链断裂 |
| 举证责任分配错误 | 本应由对方承担的举证责任分配给了你 |

**（2）二审翻盘概率** — 基于可改判错误的数量和严重程度评估

**（3）新证据价值** — 哪些一审未提交的新证据可能改变结果（引用阶段5补强方案）

**（4）再审可行性** — 如果二审也败诉，再审（审判监督程序）的可能性

**（5）上诉策略：**
- 上诉核心论点
- 建议放弃的弱论点（避免全面上诉被驳回）
- 程序性申请（如管辖异议、回避申请）
- 上诉时间窗口

## 5. 进阶技巧

### 技巧1：分段执行

案件复杂时，不要一次性跑完全部阶段。在关键节点暂停确认。

**操作方式：**
> "先执行阶段-1和阶段0，输出接案评估和法律检索后暂停等我确认。"

### 技巧2：深度追问

对红队攻击中杀伤力最高的点，要求 AI 展开更深层次的攻击推演。

**操作方式：**
> "红队对证据3的关联性攻击（攻击点2），请再展开5层攻击推演，假设对方会怎么利用这个漏洞。"

### 技巧3：模拟庭审

以对话形式模拟庭审质证环节，你扮演对方律师，AI 扮演己方律师进行对抗。

**操作方式：**
> "现在模拟庭审质证环节。你扮演我的律师，我扮演对方律师。我先问：'你方提交的微信聊天记录，如何证明聊天对象是李四本人？'"

### 技巧4：多轮迭代

根据阶段5的补强方案，实际补充证据后重新运行系统，观察胜诉概率的变化。

**操作方式：**
> "我已补充以下证据：证据5-银行流水、证据6-同事王五的证人证言。请基于更新后的证据重新推演。"

### 技巧5：针对性加强

只针对某个薄弱环节深入分析，不必重跑全流程。

**操作方式：**
> "阶段4显示证据2综合分只有30分。请专门针对证据2，给出提升采信度的3种方案。"

### 技巧6：快速路径

不需要完整推演时，直接走路径B或路径C。

**操作方式：**
> "帮我写起诉状"（路径B）或"帮我审查这份合同"（路径C）

### 技巧7：和解谈判模拟

基于阶段6的和解策略，模拟谈判过程。

**操作方式：**
> "基于阶段6的和解分析，模拟和解谈判。你扮演对方律师，我扮演我的律师。我先开价：要求对方支付5万元和解。"

## 6. 常见问题

### Q1：案件信息不够完整怎么办？

系统会在阶段-1 的接案评估中标注信息不足，并在阶段1的证据地图中标注缺失。你可以先补充缺失信息再继续，也可以让 AI 基于现有信息分析并标注"因信息不足导致的假设"。

### Q2：阶段-1 说不建议继续怎么办？

说明案件存在硬伤（如时效已过、管辖错误）。你可以：
1. **听从建议** — 节省时间，不继续分析
2. **补充信息** — 如果评估有误（如时效实际未过），补充信息后重新评估
3. **坚持继续** — 系统允许跳过门控，但会在后续分析中标注"接案评估未通过"

### Q3：红队攻击太狠了，我方真的会输吗？

红队的设计目标是**穷尽攻击路径**，不代表每个攻击都能成立。关注阶段3蓝队的成功概率评估和阶段4法官的推演结果——那才是更客观的概率判断。

### Q4：胜诉概率是准确的吗？

胜诉概率是基于当前证据和法律分析的**结构化估算**，不是精确预测。它的价值在于：
- 建立基准线（当前胜诉率是多少）
- 指导改进方向（哪些环节最值得补强）
- 衡量进步（补强后概率提升了多少）

### Q5：和解分析的结果可靠吗？

和解区间基于法官阶段的胜诉概率和补强方案的可行性计算。如果法官阶段置信度为 low，和解区间可能不准确。和解分析的价值在于**量化谈判空间**，而不是给出精确数字。

### Q6：可以用于刑事案吗？

系统设计聚焦民事诉讼和劳动仲裁。刑事案件涉及无罪推定、证据标准（排除合理怀疑）等不同规则，分析结果可能不准确。建议仅用于民事案件。

### Q7：阶段之间的数据格式是什么？

每个阶段输出 JSON 格式的结构化数据，下一阶段自动读取上一阶段的输出。你无需关心格式细节，只需关注分析内容。

### Q8：如果我对某个阶段的结论有异议？

直接告诉 AI 你的异议，要求重新分析该阶段：

> "阶段4法官给证据2的综合分只有30分，但这份证据有公证。请重新评估证据2的评分。"

## 7. 输出解读指南

### 快速判断案件健康度

完成全部阶段后，看五个数字：

| 指标 | 健康范围 | 危险信号 |
|------|---------|---------|
| 阶段-1 案件强度 `case_strength.grade` | A 或 B | D 或 E |
| 阶段4 `win_probability.applicant` | ≥ 60% | < 40% |
| 阶段7 风险评级 `risk_rating.grade` | A 或 B | D 或 E |
| 阶段5 补强方案 P0 项数量 | 0 项 | ≥ 3 项 |
| 阶段7 质量审查 `overall_reliability` | high | low |

### 优先行动顺序

1. **先看阶段-1 的接案评估** — 案件有没有硬伤，值不值得继续
2. **看阶段7 的 `core_issue_summary`** — 一句话理解本案核心问题
3. **看阶段7 的 `analysis_quality_review`** — 本次推演本身是否可靠
4. **看阶段4 的 `reasoning_chain`** — 法官如何从证据推导到结论
5. **看阶段5 的 P0 级补强方案** — 这些是必须立即做的事
6. **看阶段6 的 `settlement_zone`** — 和解空间有多大
7. **看阶段2 的交叉询问问题** — 提前准备好对方可能问的问题
8. **如果风险高，看阶段8 的上诉分析** — 输了怎么办
